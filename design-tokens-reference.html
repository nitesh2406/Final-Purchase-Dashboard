Finance ERP — Design Token Reference

Canonical Tailwind CSS classes for every reusable UI element · Last updated 2026-06-19

Golden rule: The app uses two scales — slate-* for neutrals (backgrounds, borders, text) and blue-* for interactive/brand. Status colours (emerald / red / yellow) are semantic and do not follow theme. Never use gray-* or primary-* classes anywhere in this project.

15. Rules & Conventions

Never use gray-* or primary-*. All neutral colours are slate-*. All brand/interactive colours are explicit blue-*. The custom primary- scale was removed because it had no Tailwind config backing it.
Dark mode: prefer opacity variants over opaque backgrounds. Use dark:bg-slate-800/50 not dark:bg-slate-800 for cards so the page background bleeds through subtly.
Text visibility contract: All body text in light mode must pass WCAG AA contrast on white (#fff). Minimum: text-slate-600 for secondary, text-slate-500 for labels only (not body copy). In dark mode: minimum text-slate-400 for secondary, never use text-slate-600 on dark backgrounds.
Status colours are mode-aware text only. Badge backgrounds (bg-emerald-500/10) and border opacity (border-emerald-500/20) are the same in both modes. Only the text colour changes: darker shade in light mode (text-emerald-600), lighter in dark (dark:text-emerald-400).
Hardcoded hex values are banned. Do not use bg-[#e3e3e3], bg-[#f1f1f1], bg-[#c7c7c7], text-[#000000] or similar. Always use a named Tailwind slate/blue token.
